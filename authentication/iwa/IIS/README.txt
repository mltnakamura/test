##################################################
 Windows認証のIISの設定
##################################################

※IISは認証を受けるドメインのメンバーである事が前提となります。

①IIS マネージャを開き、管理するサイトに移動します。

②[機能ビュー] の [認証] をダブルクリックします。

③[認証] ページで、[Windows 認証] を選択します。

④Windows 認証を使用にするには、[操作] ウィンドウの [有効] をクリックします。

⑤[操作] ウィンドウ　[詳細設定]
　デフォルト値のまま。

⑥[操作] ウィンドウ　[プロバイダ] 
　デフォルト値のまま。

　（参考）
	Negotiate 
		相手が Kerberos 認証をサポートしている場合は Kerberos を使用し、サポートされいなければ
		次に NTLM を使用する。
		 (Kerberos 認証に失敗したらNTLM を試みるという意味ではありません。)
 
	Negotiate : Kerberos 
		相手が Kerberos 認証をサポートしている場合は Kerberos を使用しますが、NTLM は使用しない
		(NTLM へのフォールバックはしない)
		Windows 2008 R2 からの新機能で "Negotiatable 2" または "Nego2" とも呼ばれます。
		なお、この構成にする場合は、カーネルモード認証を無効化しておく必要があります。
